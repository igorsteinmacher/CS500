# HW5 - Assignment - Unit Tests

Your friend Amy is really excited that they are learning Python! Amy just developed a simple calculator using Python, following everything they learned. Amy asked for your help to test it. With your experience, the first thing you asked was whether your friend had unit tests for the calculator; and the answer was "No".

So, you decided to help Amy by creating a set of test cases that would help find potential bugs in the code. You looked at the code, and you knew that the code has a few issues that can be caught by unit tests (despite the questionable design decisions).

In this assignment, you are asked to:
- Write, at least, 3 test cases per function, explaining what these test cases do.
- You can include tests to cover the known cases, edge cases, and possible exceptions.
- Run the tests and re-write the functions to fix the errors you find. Create a comment in the code to say what was the issue, how you caught it, and how you fixed it.
- Explore, at least, 5 different types of asserts. It is mandatory to test exception (assertRaises), at least once.
- You don't have to create tests for the functions prefixed with `print\_` and `call\_`
- You have to submit the files into a single compacted file (zip) with: the code with the fixes and comments; and the unit tests

Your friend's code is attached
